# CipherBoard 0.3.0

## English

CipherBoard 0.3.0 brings receiving and reading encrypted messages into the same
shield panel used for protected writing. Android text-action menus are no
longer required for the common flow.

### Read a copied message

1. In Telegram, SMS, email, or another transport, copy the complete message
   beginning with `CB1:`. For a multipart message, copy every part.
2. Focus a text field so CipherBoard is visible and tap the shield.
3. Select **Decrypt** at the top of the embedded panel.
4. Tap **Paste and decrypt**. CipherBoard explicitly reads one bounded clipboard
   item, validates it as a complete CipherBoard message, and leaves the
   ciphertext clip unchanged.
5. Unlock the Vault when requested.
6. Read the plaintext directly in the protected keyboard panel. The ordinary
   keys are hidden in Decrypt mode, and the message cannot be selected or
   copied.
7. Tap **Reply securely** to hide the received plaintext, return to **Encrypt**,
   and select the same local contact.

The existing **Decrypt in CipherBoard** Android text action and separate secure
viewer remain available in transports that expose text actions. Both paths keep
plaintext inside CipherBoard and never return it to the transport.

### Security behavior

- The embedded panel remains inside the IME's `FLAG_SECURE` window. Plaintext is
  rendered by a drawing-only surface with no focus, text selection, long-click
  action mode, saved state, Accessibility text, autofill, or content capture.
- Decrypt mode hides the standard keyboard keys and suggestions. It does not
  offer a plaintext input or host-commit path.
- Clipboard access occurs only after **Paste and decrypt** and accepts exactly
  one bounded text item. CipherBoard does not replace or clear the copied
  ciphertext and never writes decrypted plaintext to the clipboard.
- Vault authentication uses a random process-local, one-shot handoff to the
  non-exported unlock activity. Its Intent contains the handoff token only, not
  ciphertext or plaintext. Duplicate, cancelled, timed-out, or mismatched
  handoffs fail closed.
- The inbound ratchet, replay marker, and encrypted pending-display record are
  committed atomically before plaintext reaches the surface. The display lease
  is acknowledged only after the first allowed draw. If drawing never occurs,
  the surface clears and the encrypted record remains available for exact crash
  recovery without decrypting the Olm message again.
- After a normal acknowledged display is hidden or closed, its pending record
  is removed under the default no-plaintext-history policy. A process death
  after first draw but before close can leave that encrypted record recoverable
  during the bounded recovery window; this is not a strict exactly-once viewer.

### Layout and validation

The embedded panel now has constrained-height branches for landscape and large
font scales, a bounded scrollable plaintext region, and minimum-size command
controls. The release QA matrix explicitly covers English and Russian Encrypt
and Decrypt states in portrait, landscape, and font scale 2.0. Automated
surface tests cannot replace visual checks on the exact release APK, and
physical GrapheneOS validation remains required before high-risk use.

Targeted JVM/Robolectric tests pass for the ciphertext-only clipboard reader,
drawing-only surface and first-draw callback, one-shot unlock bridge,
already-unlocked backend gate, background cancellation, owned worker-result
handoff, reply draft wipe, secure-IME lifecycle behavior, and the legacy
clipboard Activity. The API 36 native test passes 1/1 and the Android security
instrumentation suite passes 7/7. Manual API 36 checks confirm the locked
Encrypt and idle Decrypt layouts fit in English and Russian landscape at font
scale 2.0; paired-contact and physical GrapheneOS validation remain. The
complete application has not received an independent applied-cryptography or
Android security audit.

Install `CipherBoard-0.3.0-release.apk` over the existing release. Do not
uninstall first: uninstalling destroys the Vault, local identity, contacts, and
ratchet state. This release does not intentionally change the `CB1` protocol,
application ID, signing identity, or Vault schema. Verify the published
SHA-256 and signing-certificate fingerprint, and stop if Android reports a
signer mismatch.

CipherBoard has no in-app updater and requests neither Internet nor package-
installation permission. For optional update notifications, configure the
external Obtainium app with `https://github.com/bglglzd/CipherBoard`, disable
pre-releases, and use this release asset filter:

```text
^CipherBoard-[0-9]+\.[0-9]+\.[0-9]+-release\.apk$
```

Obtainium is a separate, networked trust boundary. CipherBoard itself remains
offline.

## Русский

В CipherBoard 0.3.0 получение и просмотр зашифрованных сообщений перенесены в
ту же панель со щитом, которая используется для защищённого набора. Для
основного сценария больше не требуется контекстное меню Android.

### Как прочитать скопированное сообщение

1. В Telegram, SMS, электронной почте или другом транспорте скопируйте целиком
   сообщение, начинающееся с `CB1:`. Для составного сообщения нужны все части.
2. Откройте любое поле ввода, чтобы появилась CipherBoard, и нажмите щит.
3. Выберите сверху режим **Расшифровать**.
4. Нажмите **Вставить и расшифровать**. CipherBoard явно прочитает один
   ограниченный по размеру элемент буфера, проверит формат полного сообщения и
   оставит скопированный шифротекст без изменений.
5. При запросе разблокируйте Vault.
6. Прочитайте открытый текст прямо в защищённой панели клавиатуры. Обычные
   клавиши в режиме расшифровки скрыты, а сообщение нельзя выделить или
   скопировать.
7. Нажмите **Ответить защищённо**: полученный открытый текст будет скрыт, панель
   переключится в режим **Зашифровать** и выберет того же локального контакта.

Системное действие **Расшифровать в CipherBoard** и отдельный защищённый
просмотрщик сохранены для приложений, которые показывают Android text actions.
В обоих сценариях открытый текст остаётся внутри CipherBoard и не возвращается
во внешний транспорт.

### Защитные свойства

- Встроенная панель находится в окне IME с `FLAG_SECURE`. Открытый текст
  рисуется специальной поверхностью без фокуса, выделения, контекстных действий,
  сохранения состояния, текста для Accessibility, автозаполнения и content
  capture.
- В режиме расшифровки скрыты обычные клавиши и подсказки. Этот экран не
  предоставляет путь для ввода открытого текста во внешнее поле.
- CipherBoard обращается к буферу только после нажатия **Вставить и
  расшифровать** и принимает ровно один текстовый элемент ограниченного размера.
  Скопированный шифротекст не заменяется и не очищается. Расшифрованный текст
  никогда не записывается в буфер обмена.
- Для разблокировки Vault используется случайный одноразовый токен внутри
  процесса, передаваемый закрытой Activity. В Intent нет шифротекста или
  открытого текста. Повторный, отменённый, просроченный или не соответствующий
  активному полю переход завершается безопасным отказом.
- Состояние ratchet, replay-маркер и зашифрованная временная запись просмотра
  фиксируются атомарно до передачи открытого текста в UI. Запись считается
  показанной только после первой разрешённой отрисовки. Если отрисовка не
  произошла, поверхность очищается, а зашифрованная запись остаётся для точного
  восстановления без повторной расшифровки Olm-сообщения.
- После обычного показа и скрытия временная запись удаляется согласно режиму
  без истории открытого текста. Если процесс завершится после первой отрисовки,
  но до закрытия поверхности, зашифрованная запись может сохраниться на
  ограниченное время восстановления. Это не строгий просмотр exactly-once.

### Адаптивность и проверки

Для панели добавлены ограниченные по высоте варианты для альбомной ориентации и
крупных системных шрифтов, прокручиваемая область открытого текста и минимальные
размеры команд. Матрица проверки релиза явно включает английский и русский
интерфейс режимов отправки и чтения в портретной и альбомной ориентации, а также
при масштабе шрифта 2.0. Автотест поверхности не заменяет визуальную проверку
точного release APK; перед использованием в ситуации высокого риска всё ещё
необходима физическая проверка на GrapheneOS.

Целевые JVM/Robolectric-тесты проходят для чтения только шифротекста из буфера,
защищённой поверхности и первой отрисовки, одноразового перехода разблокировки,
слоя расшифровки без собственного окна аутентификации, отмены при уходе в фон,
владения результатами фоновых задач, очистки чужого черновика при ответе,
жизненного цикла защищённой IME и прежнего резервного экрана буфера обмена.
Native-тест на API 36 проходит 1/1, Android instrumentation проходит 7/7.
Ручная проверка API 36 подтверждает, что заблокированный режим отправки и
пустой режим чтения помещаются в английском и русском landscape при масштабе
шрифта 2.0; сценарий с сопряжённым контактом и физическая GrapheneOS всё ещё
требуют проверки. Полное приложение не проходило независимый аудит прикладной
криптографии и Android security.

Установите `CipherBoard-0.3.0-release.apk` поверх текущей release-версии. Не
удаляйте приложение: удаление уничтожит Vault, локальную identity, контакты и
состояние ratchet. Этот релиз намеренно не меняет протокол `CB1`, application
ID, ключ подписи или схему Vault. Проверьте опубликованные SHA-256 и fingerprint
сертификата. При сообщении Android о несовпадении подписи остановитесь.

В CipherBoard нет встроенного обновления и разрешений на Internet или установку
пакетов. Для необязательных уведомлений об обновлениях настройте внешнее
приложение Obtainium на `https://github.com/bglglzd/CipherBoard`, отключите
pre-release и используйте фильтр release-файлов:

```text
^CipherBoard-[0-9]+\.[0-9]+\.[0-9]+-release\.apk$
```

Obtainium использует сеть и является отдельной границей доверия. Сам CipherBoard
остаётся полностью офлайн.
